# ЯДРО СУБЪЕКТА · ЖИВОЙ ЦИКЛ

Стратегическая браузерная игра становления, собранная из архитектоники проекта «Гдея» и логики ТзАрх.

**Публичный адрес после активации GitHub Pages:** `https://lacinnik.github.io/Game-GDEYA/`

Игра подготовлена к публикации через GitHub Pages и после активации открывается в Safari, Chrome и других браузерах без входа в ChatGPT. На iPhone её можно добавить на экран «Домой» из меню «Поделиться» Safari.

## Платформа и опубликованные вертикали

- **Platform 2.0:** `https://lacinnik.github.io/Game-GDEYA/platform/` — единый реестр 22 сущностей и переходы между лабораториями ГДЕЯ и РЕЗОН.
- **MODULE 1.0 stable:** `https://lacinnik.github.io/Game-GDEYA/labs/module/` — исполнимый контур `49 Азов × 24 Буки × 7 Передач` с локальным паспортом.
- **VoidOCR 1.0 stable:** `https://lacinnik.github.io/Game-GDEYA/labs/voidocr/` — локальный четырёхэтапный протокол различения с JSON-экспортом и подготовленным офлайн-контуром.

Реестр является источником статусов для лаунчера. Статусы дисциплинарного корпуса ТзАр сверяются с `contour/manifest.json` репозитория `Lacinnik/-tensor-architectonics`; QENGINE публикуется только как `candidate · author-reviewed`, не как канон.

## Замысел

Игрок проводит живое намерение через четыре плиты:

**Ресурс → Власть → Отношения → Результат**

Цель — материализовать намерение, сохранив инвариант и не отдав ядро градиенту поля.

Каждый ход проходит четыре такта:

1. **Наблюдение** — различить точное число состояний, заданное картой.
2. **Конфигурация** — выбрать геометрию и тип точки.
3. **Формула** — собрать переход `Аз × Бука → Передача`.
4. **Отклик поля** — получить изменение метрик и фазы Источника.

## Что вошло в игру

- 96 карт: 4 плиты × 6 процессов × 4 фазы;
- быстрый цикл на 4 узла и полный канонический цикл на 24 узла;
- 5 геометрий, 4 типа точек, 49 Азов, 24 Буки и 7 Передач;
- автоматический расчёт `α`, `IЯ`, `Cₘ`, `Q` и `T`;
- слои Тело → Эмоции → Мысль → Творец;
- аварийный тактический контур и события `M⁷–M¹²`;
- итоговая когерентность и экспорт чертежа прохождения в JSON;
- голосовой проводник Скеллу: озвучивание шага, повтор инструкции и контекстная диктовка;
- адаптивный интерфейс с сенсорным управлением для iPhone.

## Ручная проверка Safari на iPhone

- Открой игру по HTTPS, разверни Скеллу и нажми «Включить ведение»: первый текст должен прозвучать только после этого нажатия.
- Нажми «Повторить шаг», затем «Выключить голос»: речь и световой статус должны остановиться сразу.
- Включи ведение и пройди на следующий шаг: новая фраза заменяет предыдущую, без двойной речи.
- Нажми контекстную диктовку, разреши микрофон и произнеси одну короткую мысль: она попадает в активное поле, статус возвращается из «слушает».
- Пока Скеллу слушает, повторно нажми кнопку: слушание останавливается; повторное нажатие запускает новую попытку.
- Отклони доступ к микрофону или используй Safari без распознавания: кнопка фокусирует нужное поле для системной диктовки клавиатуры.

## Локальный запуск

Требуется Node.js `22.13+`.

```bash
npm ci
npm run dev
```

После запуска откройте адрес, указанный Vite в терминале.

## Проверка

```bash
npm test
```

Команда собирает проект, валидирует артефакт и запускает 11 автоматических проверок игры, Platform 2.0, MODULE, VoidOCR и его офлайн-оболочки. Публикация Pages дополнительно требует успешного lint и публичной Vite-сборки.

Сквозная матрица повышения VoidOCR и Collective Field Check до stable зафиксирована в [`docs/PLATFORM_2_0_STABLE_ACCEPTANCE_2026-07-22.md`](docs/PLATFORM_2_0_STABLE_ACCEPTANCE_2026-07-22.md).
